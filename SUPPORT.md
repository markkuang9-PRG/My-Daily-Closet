# Support

Use the right path for the right kind of request so the repository stays useful for contributors and the project owner can handle sensitive matters privately.

## Public project support

Use public GitHub issues for:

- reproducible bugs;
- feature requests;
- documentation fixes;
- general implementation questions that can be discussed in public.

## Private or sensitive matters

Do not open a public issue for:

- security vulnerabilities;
- private business proposals;
- trademark permissions;
- licensing exceptions;
- confidential partnership or acquisition discussions.

For those matters, contact the project owner privately through GitHub.

## Commercial licensing

If you found a security issue, follow [SECURITY.md](SECURITY.md).
