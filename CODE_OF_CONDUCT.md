# Code of Conduct

## Our Standard

We want My Daily Closet to be a productive, respectful project for contributors, users, maintainers, and observers.

Examples of behavior that help create that environment include:

- being respectful in feedback, reviews, and technical disagreement;
- assuming good intent while still being precise about quality and evidence;
- welcoming contributors from different backgrounds and skill levels;
- focusing criticism on code, product decisions, and documented behavior rather than on people;
- sharing credit fairly.

Examples of unacceptable behavior include:

- harassment, intimidation, threats, or discrimination;
- personal attacks, insults, or sustained hostility;
- doxxing or sharing private information without permission;
- spam, trolling, or intentionally disruptive behavior;
- knowingly submitting malicious code, harmful payloads, or confidential material.

## Scope

This Code of Conduct applies in the repository, issues, pull requests, discussions, documentation, and project-related community spaces controlled by the project owner.

## Enforcement

Project maintainers may remove, edit, reject, or moderate content and participation that they believe violates this Code of Conduct.

Possible responses include:

- a warning;
- content removal;
- temporary restrictions;
- permanent removal from project spaces.

## Maintainer Discretion

Interpretation and enforcement are at the sole discretion of the project maintainers and project owner.

## Reporting

If you experience or witness unacceptable behavior, report it privately to the project owner through a private GitHub contact path rather than filing a public issue if privacy is needed.
